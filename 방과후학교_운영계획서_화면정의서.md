# 방과후학교 운영계획서 - 화면 정의서

## 1. 화면 개요

### 1.1 목적
- 학교 방과후학교 운영계획서를 AI 기반으로 자동 생성
- 필수 항목 누락 방지 및 표준 양식 준수
- 사용자 입력 최소화 및 정확도 극대화

### 1.2 화면 구성 원칙
- **구조화된 입력**: 자유 서술 최소화, 선택지/표 중심
- **단계별 진행**: 기본정보 → 프로그램 → 운영방식 → 안전관리
- **실시간 검증**: 입력 즉시 필수값/형식 체크
- **미리보기 제공**: 문서 생성 전 입력 내용 확인

---

## 2. 화면 흐름도

```
[문서 유형 선택] 
    ↓
[기본 정보 입력]
    ↓
[운영 목표/방침 설정]
    ↓
[프로그램 운영표 작성] ← 핵심 화면
    ↓
[모집·수납·회계 설정]
    ↓
[안전·학생 관리 설정]
    ↓
[입력 내용 미리보기]
    ↓
[문서 생성 및 다운로드]
```

## 3. 화면별 상세 정의

### 3.1 기본 정보 입력 화면

#### 화면 레이아웃
```
┌─────────────────────────────────────┐
│ 📋 방과후학교 운영계획서 작성       │
├─────────────────────────────────────┤
│ 1단계: 기본 정보                    │
│                                     │
│ 학교명*         [______________]    │
│ 학년도*         [2025▼]             │
│ 학기*           ○ 1학기 ○ 2학기    │
│                 ○ 연중              │
│                                     │
│ 문서 제목       [자동생성됨]         │
│                 (예: 2025학년도 1학기│
│                  방과후학교 운영계획)│
│                                     │
│ 운영 기간*                          │
│   시작일       [2025-03-01]         │
│   종료일       [2025-08-31]         │
│                                     │
│ 운영 요일*     □월 □화 □수 □목 □금│
│                                     │
│ 1회 운영시간*  [___]시간 [___]분   │
│                                     │
│ 담당 부서      [______________]     │
│ 담당자         [______________]     │
│ 문의처         [______________]     │
│                                     │
│        [이전]          [다음 →]     │
└─────────────────────────────────────┘
```

#### 입력 항목 명세

| 항목명 | 유형 | 필수 | 설명 | 검증 규칙 |
|--------|------|------|------|-----------|
| 학교명 | 텍스트 | ✓ | 학교 정식 명칭 | 2~30자 |
| 학년도 | 드롭다운 | ✓ | 현재년도±2년 범위 | 숫자 4자리 |
| 학기 | 라디오버튼 | ✓ | 1학기/2학기/연중 | 단일 선택 |
| 문서 제목 | 자동생성 | - | 학년도+학기+문서유형 | 읽기전용 |
| 운영기간(시작) | 날짜선택 | ✓ | 운영 시작일 | YYYY-MM-DD |
| 운영기간(종료) | 날짜선택 | ✓ | 운영 종료일 | 시작일 이후 |
| 운영 요일 | 체크박스 | ✓ | 월~금 중 선택 | 1개 이상 |
| 1회 운영시간 | 숫자입력 | ✓ | 시간, 분 | 시간:0~24, 분:0~59 |
| 담당 부서 | 텍스트 | - | 예: 교무부, 학생부 | 1~20자 |
| 담당자 | 텍스트 | - | 담당 교사명 | 1~10자 |
| 문의처 | 텍스트 | - | 전화번호 또는 이메일 | 형식 검증 |

## 4. 공통 UI 요소

### 4.1 진행 상태 표시
```
1 기본정보 → 2 운영목표 → 3 프로그램 → 4 모집수납 → 5 안전관리 → 6 확인
●━━━━━━━○━━━━━━━○━━━━━━━○━━━━━━━○━━━━━━━○
```

### 4.2 필수 항목 표시
- `*` 마크 또는 빨간색 표시
- 미입력 시 다음 단계 진행 차단

### 4.3 도움말 툴팁
- `ⓘ` 아이콘 호버 시 설명 표시
- 예: "수강료는 1인당 전체 기간 총액을 입력하세요"

### 4.4 에러 메시지
```
⚠️ 운영 종료일은 시작일 이후여야 합니다.
⚠️ 프로그램을 최소 1개 이상 등록해주세요.
✅ 모든 필수 항목이 입력되었습니다.
```

## 5. 반응형 디자인

### 5.1 데스크톱 (1200px 이상)
- 2단 레이아웃 가능
- 프로그램 테이블 전체 표시

### 5.2 태블릿 (768px ~ 1199px)
- 1단 레이아웃
- 프로그램 카드형 표시

### 5.3 모바일 (768px 미만)
- 스택형 레이아웃
- 입력 항목 세로 배치
- 하단 고정 버튼

---

## 6. 접근성

- 키보드 네비게이션 지원 (Tab, Enter)
- 스크린 리더 호환 (aria-label)
- 명도 대비 4.5:1 이상
- 폰트 크기 최소 14px
